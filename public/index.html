Informative Bulk Insert Details

Before executing the bulk insertion, the app should inform me the following information

  • Which module does not exists and will be newly created?
  • Which module exists and values will be updated?
  • Which module exists and will not be updated?

  • Once acknowledged by user, go ahead and insert/update the entries
  • At most another 5 HTTP request and 5 SQL queries (even for 100 new modules)
